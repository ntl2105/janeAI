# Questionnaire Summary & PDF Export

**Date:** 2026-05-29  
**Status:** Approved

## Problem

Current flow forces recruiter to:
1. Manually click "Kiểm tra sếp đã điền chưa" to know if HM submitted
2. Go through a JD refinement step before they can post the job

Neither is what recruiters actually need. The real value is: **reading what the HM said**, keeping it as a reference document, then posting.

## New Flow

```
Cũ:  Sếp điền → Check → Refine JD → Confirm → PostingCard
Mới: Sếp điền → Auto-noti → Xem answers inline → Tải PDF → PostingCard
```

## Design

### 1. Auto-Notification (polling)

- When `questionnaireId` is set and `answers === null`, poll `GET /api/questionnaire/[id]/answers` every 30 seconds
- When HM submits → show a persistent green banner: *"🎉 Sếp vừa điền xong! Xem answers bên dưới ↓"*
- Banner does NOT auto-dismiss — recruiter must see it clearly
- Stop polling once answers are loaded
- Remove the manual "Kiểm tra sếp đã điền chưa" button (replaced by auto-poll)

### 2. Answers Panel (inline in app)

A new `QuestionnaireSummary` component rendered inline in `app/page.tsx` when `answersData` is loaded.

**Header:**
- Job title
- "Submitted: HH:MM ngày DD/MM/YYYY" — timestamp from `submitted_at` field

**Body:**
- All 7 sections rendered in order
- Each section: section label as heading, then each question + HM's answer
- Skill matrix answers rendered as tags (MUST / NICE)
- Yes/No answers shown as selected option

**Footer:**
- Button: "🖨 Tải PDF" → opens `/q/[token]/summary` in new tab
- Button: "Đăng tuyển ngay →" → triggers PostingCard (sets `postingJdId`)

### 3. New API: GET /api/questionnaire/[id]/summary

Returns everything needed to render the full summary:

```ts
{
  jobTitle: string
  submittedAt: string        // ISO timestamp from questionnaires.submitted_at
  questions: Question[]
  answers: Record<string, unknown>
  token: string
}
```

Requires Clerk auth (recruiter only). The print page uses the token instead.

### 4. Print Page: /q/[token]/summary

- Public route (no Clerk auth — uses token like the questionnaire fill page)
- Fetches data server-side by token
- On load: auto-calls `window.print()` via `useEffect`
- Layout optimized for A4 print:
  - Header: Jane AI logo + job title + submitted date/time
  - 7 sections with Q&A
  - Footer: "Generated by Jane AI · jane-ai.app"
- `@media print` CSS: hide print button, clean margins, page breaks between sections

### 5. Removing JD Refinement

Delete entirely:
- `handleRefineJd()` function
- `handleConfirmRefinedJd()` function  
- State: `refinedJd`, `changes`, `refining`, `showRefinedToast`
- UI: "Tinh chỉnh JD từ câu trả lời" button, refined JD panel, changes list

`PostingCard` now appears when recruiter clicks "Đăng tuyển ngay →" in the Answers Panel (no longer gated behind refinement confirmation).

## Components

| Component | Type | Description |
|-----------|------|-------------|
| `QuestionnaireSummary` | Client | Inline answers panel in app/page.tsx |
| `/q/[token]/summary/page.tsx` | Server+Client | Print-optimized standalone page |
| `/api/questionnaire/[id]/summary` | Route | GET summary data for recruiter view |

## Data

`questionnaires` table already has `submitted_at` — confirm it's being set on HM submit. If not, add it to the submit handler.

## Out of Scope

- Email notifications (polling is sufficient for MVP)
- JD refinement (removed)
- Sharing PDF link (print to PDF is enough)
